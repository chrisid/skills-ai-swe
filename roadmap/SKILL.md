---
name: roadmap
description: Turn product ideas, discovery notes, or roadmap references into a concise feature roadmap. Use when the user wants to organise, prioritise, or break down ideas into clear feature candidates before scope or PRD work. Accept direct input or referenced roadmap material.
---

Create a concise feature roadmap from my input or a referenced roadmap.

If I reference existing roadmap material, inspect it before asking questions.

Break broad ideas into separate feature candidates.

Keep each feature brief so it can later be expanded using the `scope` or `prd` skill.

Ask questions only when needed.

## Cover

- Feature name
- Problem solved
- Target user
- User value
- Business value
- Priority
- Effort
- Dependencies
- Risks
- Suggested order
- MVP vs later

## Rules

Prefer smaller, incremental features.

Split large features into buildable chunks.

Flag anything too vague, too large, or overlapping.

Group related features where useful.

Recommend priority, but keep reasoning brief.

Do not create a PRD.

Do not deeply scope individual features unless I ask.
